## 二、实验设计

### 实验一：密封数据的机密性与完整性

1. 检查密封文件中是否存在密码、用户名、服务名称明文
2. 修改密封文件中的单个字节
3. 随机覆盖部分内容
4. 截断密封文件
5. 修改文件头或文件尾
6. 拼接或追加额外数据
7. 记录解封结果、错误类型和程序行为

### 实验二：`MRENCLAVE` 与 `MRSIGNER` 对比

1. 准备 Enclave V1
2. 准备代码发生变化的 Enclave V2
3. 使用 `MRENCLAVE` 策略生成密封文件
4. 使用 `MRSIGNER` 策略生成密封文件
5. V1 解封 V1 数据
6. V2 解封 V1 的 `MRENCLAVE` 数据
7. V2 解封 V1 的 `MRSIGNER` 数据
8. 比较安全绑定强度和升级兼容性

### 实验三：回滚攻击与防护

1. 生成旧状态文件 `vault_v1.sealed`
2. 修改密码库并生成新状态文件 `vault_v2.sealed`
3. 使用旧文件替换当前文件
4. 验证仅使用 Sealing 时旧文件仍可解封
5. 引入可信版本记录
6. 比较密封文件中的 `state_version` 与可信版本
7. 检测并拒绝旧版本文件
8. 对比无防护和有防护时的结果
